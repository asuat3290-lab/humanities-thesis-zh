# 逐章修改、全文统稿与版本管理

## 一、先判定修改级别

| 级别 | 允许操作 |
|---|---|
| 诊断 | 评价功能、逻辑、概念、证据、脚注和语言，不改正文 |
| 方案 | 指定保留、合并、删除、移动、补证和改写位置 |
| 小修 | 句子调整、段落缝合、脚注移动、轻微删重 |
| 局部重写 | 重写指定段落或小节，保持章节主线 |
| 章节重写 | 重新组织整章，需要用户明确授权 |
| 全文统稿 | 合并正式章稿，处理跨章关系和统一交付 |

用户给出的级别是上限，不因发现更多问题自行扩大范围。

## 二、修改前诊断

对指定范围回答：

1. 它在全文中承担什么功能。
2. 当前段落是否真正完成这一功能。
3. 同上一章和下一章的接口是否明确。
4. 哪些判断重复，哪些只是必要复指。
5. 概念是否滑移或未经解释。
6. 哪些段落“看似顺滑但没有推进”。
7. 原始文本、二手研究或现实材料是否不足。
8. 脚注是否过宽、错位或无法核验。
9. 语言是否存在模板化和跳跃。

用户要求“先诊断”“先拿方案”时，到此停止。

## 三、局部修改的优先顺序

依次尝试：

1. 删除无功能句和加工痕迹。
2. 合并表达相同判断的相邻句。
3. 补充必要的主语、限定和中介句。
4. 调整句序，使证据紧跟判断。
5. 移动脚注到真正受支持的句子后。
6. 合并重复段，保留证据更充分的一段。
7. 重写无法通过局部调整修复的段落。

不要一开始就整段重写。保留作者已有的有效表达和论证痕迹。

## 四、处理重复

重复分为三类：

- **无功能重复**：同一判断、证据和结论再次出现，应删除或合并。
- **接口复指**：新章开头简要承接前章结论，可以保留，但不得重新证明。
- **递进重现**：同一概念进入新的分析层次，需要明确增加了什么规定。

检查第一章和第二章时尤其注意：绪论提出问题和研究现状，理论章负责概念限定和论证，不能各写一遍完整概念史。

检查相邻理论章时，区分成立条件、运行机制、历史限度和当代变化。后章只继承前章结论，不从头重建。

## 五、控制篇幅

篇幅是论证结果，不是机械配额。

- 预期字数已达到但论证未完成，可以继续写。
- 字数较长不等于必须压缩；先判断是否重复。
- 一节显著长于其他节时，先检查它是否承担过多任务。
- 拆节只有在两个问题可以分别形成输入和输出时才成立。
- 为保持连贯允许适量解释，但不能用同义反复填充篇幅。

## 六、章节接口

章末不写“下一章将讨论”。用尚未解决的问题自然过渡：

- 本章已确立什么；
- 这一结论尚不能解释什么；
- 继续解释需要进入哪一组现实关系。

下一章开头只用一段承接，不重复前章的完整论证。

## 七、全文合并

合并前列出各章唯一正式版本，禁止混用旧稿。合并后依次处理：

1. 题目、摘要、目录、正文、结论和参考文献是否齐全。
2. 章节标题和目录是否一致。
3. 核心概念是否统一。
4. 外语原词和外国人名是否只在首次实质出现处标注。
5. 跨章重复是否属于必要接口。
6. 摘要是否呈现问题、结构、批判根据和当代效力。
7. 结论是否集中回答核心问题，而非逐章摘要。
8. 研究限度是否是学术限制，不保留制作过程说明。

## 八、版本命名

采用可追踪文件名：

“作者-论文题目或章节-修订性质-YYYY-MM-DD.ext”

例如：

“某某-硕士学位论文-第三章-概念校准版-2026-07-15.md”

保留源文件，输出到明确目录。不要使用“final-final2”“最新版新”等无法追踪的命名。

## 九、修改说明

最终说明只列：

- 修改类型；
- 关键概念或论证变化；
- 文件链接；
- 验证结果；
- 仍待人工核验事项。

不要在最终回复中粘贴完整论文，也不要把内部审查过程写入正文。
